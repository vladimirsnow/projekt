# Hexlet Messenger

`Hexlet Messenger` — realtime веб-мессенджер в стиле Telegram.

## Что реализовано

- Личные чаты (1:1)
- Групповые чаты
- Публичные каналы
- Роли админов в группах и каналах
- Управление участниками группы/канала (исключение, назначение/снятие админа)
- Регистрация и вход по нику/паролю
- Восстановление пароля по email (через ссылку/код из письма)
- Поиск пользователей по нику для старта личного чата
- Настройки аккаунта (смена ника, email, пароля, аватара)
- Выход из аккаунта
- Отправка текстовых сообщений в реальном времени (Socket.IO)
- Передача медиа и файлов (фото, видео, аудио и другие файлы)
- Предпросмотр вложений перед отправкой
- Просмотр/проигрывание медиа в сообщениях
- Аудио и видеозвонки (WebRTC + сигналинг через Socket.IO)
- Скрытый список зарегистрированных пользователей (личный чат только по известному нику)
- Каналы появляются в списке чатов только после подписки
- В каналах писать могут только админы/создатель
- Поиск по чатам
- Тёмная тема интерфейса
- Адаптивность: desktop/tablet/mobile, мобильное меню чатов

## Технологии

- Node.js + Express
- Socket.IO
- WebRTC
- bcryptjs (хеширование паролей)
- Nodemailer (email-восстановление)
- Multer (загрузка файлов)
- HTML/CSS/Vanilla JS

## Запуск

1. Установи зависимости:

```bash
npm install
```

2. Настрой окружение (особенно SMTP для восстановления пароля):

```bash
cp .env.example .env
```

3. Запусти сервер:

```bash
npm start
```

4. Открой в браузере:

```text
http://localhost:3000
```

## Доступ из локальной сети (много пользователей одновременно)

- Сервер по умолчанию слушает `0.0.0.0`, поэтому к нему можно подключаться с других устройств в той же сети.
- После запуска в консоли выводятся LAN-адреса вида `http://192.168.x.x:3000`.
- Открывай этот адрес на других телефонах/ноутбуках, чтобы тестировать одновременную работу пользователей.

## Авторизация и безопасность

- У каждого пользователя отдельный аккаунт: `ник + email + пароль`.
- Вход только по правильному паролю, просто по нику войти нельзя.
- Пароли хранятся в виде bcrypt-хеша (`data/users.json`).
- Личный чат создается только по известному нику пользователя.
- Список всех зарегистрированных пользователей в интерфейсе не показывается.

## Восстановление пароля по email

- В форме входа есть ссылка `Забыли пароль?`.
- После нажатия открывается отдельное окно с вводом email.
- На почту приходит письмо со ссылкой и кодом восстановления.
- Затем этим кодом можно задать новый пароль.
- Срок действия кода: 15 минут.

### SMTP (боевой режим)

Для реальной отправки писем задай переменные окружения:

- `SMTP_HOST`
- `SMTP_PORT`
- `SMTP_SECURE=true|false`
- `SMTP_USER`
- `SMTP_PASS`
- `SMTP_SERVICE` (альтернатива `SMTP_HOST`, например `gmail`)
- `MAIL_FROM` (опционально)

Если SMTP не настроен, сервер не сможет отправлять письма восстановления и вернет ошибку конфигурации.

## Звонки и доступ к микрофону

- Браузеры дают доступ к микрофону/камере только в безопасном контексте: `https://...` или `localhost`.
- На LAN-адресе вида `http://192.168.x.x:3000` звонки могут быть заблокированы политикой браузера.
- Для LAN-звонков включай HTTPS:

  - `SSL_KEY_PATH=/path/to/key.pem`
  - `SSL_CERT_PATH=/path/to/cert.pem`

## Как тестировать как "реальный" мессенджер

- Открой приложение в двух разных браузерах или в двух вкладках в режиме инкогнито.
- Войди под разными именами пользователей.
- Создай личный чат/группу/канал и отправляй сообщения.
- Проверь загрузку фото/видео/аудио, предпросмотр и воспроизведение.
- Проверь аудио и видеозвонок между двумя пользователями.

## Структура проекта

- `server.js` — сервер, Socket.IO события, upload endpoint, сигналинг WebRTC.
- `public/index.html` — интерфейс мессенджера.
- `public/styles/main.css` — стили UI.
- `public/scripts/main.js` — клиентская логика (чаты, сообщения, медиа, звонки).
- `uploads/` — загруженные файлы (создается автоматически).
